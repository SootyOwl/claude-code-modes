# Design: Single-Install Distribution

## Overview

This design transforms claude-code-modes from a clone-to-use repo into a distributable package via npm (`bunx claude-code-modes create`) and compiled Bun binaries (GitHub Releases with curl-pipe-bash installer). The three core changes: (1) embed built-in prompt fragments into a generated TypeScript map, (2) replace the bash wrapper with TypeScript-native TTY handoff via `Bun.spawn`, (3) add build/release/install infrastructure.

## Implementation Units

### Unit 2: Refactor `assemble.ts` to Use Embedded Prompts

**File**: `src/assemble.ts`

```typescript
import { EMBEDDED_PROMPTS } from "./embedded-prompts.js";

/**
 * Resolves a fragment's content. Built-in fragments (relative paths) are
 * looked up in the embedded map. Custom fragments (absolute paths) are
 * read from disk.
 */
function resolveFragment(fragPath: string): string | null
```

**Current code** (`assemblePrompt` loop, lines 111-128):
```typescript
for (const fragPath of fragmentPaths) {
  const fullPath = isAbsolute(fragPath)
    ? fragPath
    : resolve(promptsDir, fragPath);
  let content: string | null;
  try { content = readFileSync(fullPath, "utf8"); } catch { content = null; }
  if (content === null) throw new Error(`Missing prompt fragment: ${fragPath}`);
  sections.push(content.trim());
}
```

**New code**:
```typescript
for (const fragPath of fragmentPaths) {
  let content: string | null;
  if (isAbsolute(fragPath)) {
    // Custom fragment — read from disk
    try { content = readFileSync(fragPath, "utf8"); } catch { content = null; }
  } else {
    // Built-in fragment — check embedded map first, fall back to disk
    content = EMBEDDED_PROMPTS[fragPath] ?? null;
    if (content === null) {
      const fullPath = resolve(promptsDir, fragPath);
      try { content = readFileSync(fullPath, "utf8"); } catch { content = null; }
    }
  }
  if (content === null) throw new Error(`Missing prompt fragment: ${fragPath}`);
  sections.push(content.trim());
}
```

**Implementation Notes**:
- The embedded map lookup is the fast path for built-in fragments — no filesystem I/O
- Disk fallback for relative paths preserves dev workflow (editing prompts without regenerating)
- `promptsDir` parameter is still needed for the disk fallback path
- `readFragment()` export is also updated to check embedded map first (used by `inspect.ts`)
- The `AssembleOptions` interface keeps `promptsDir: string` — it's still used for disk fallback and by inspect

**Acceptance Criteria**:
- [ ] `assemblePrompt()` resolves built-in fragments from `EMBEDDED_PROMPTS` without disk reads
- [ ] Custom fragments (absolute paths) still read from disk
- [ ] Relative paths not in embedded map fall back to disk (dev convenience)
- [ ] All existing assemble tests pass without modification
- [ ] `readFragment()` also checks embedded map first

### Unit 3: Replace Bash Wrapper with TypeScript Entry Point

**File**: `src/cli.ts` (new file — the compiled binary's entry point)

```typescript
#!/usr/bin/env bun
/**
 * CLI entry point that replaces the bash wrapper.
 * For non-interactive commands (--help, --print, config, inspect):
 *   runs build-prompt logic directly and exits.
 * For interactive commands:
 *   assembles the prompt, spawns `claude` with inherited stdio, and
 *   waits for it to exit.
 */
import { join } from "node:path";
import { parseCliArgs } from "./args.js";
import { loadConfig } from "./config.js";
import { resolveConfig } from "./resolve.js";
import { assemblePrompt, writeTempPrompt } from "./assemble.js";
import { detectEnv, buildTemplateVars } from "./env.js";
import { runConfigCommand } from "./config-cli.js";
import { runInspectCommand } from "./inspect.js";

function printUsage(): void
// Same usage text as current build-prompt.ts printUsage()

async function main(): Promise<void> {
  const argv = process.argv.slice(2);

  // No args or --help: show usage and exit
  if (argv.length === 0 || argv.includes("--help") || argv.includes("-h")) {
    printUsage();
    process.exit(0);
  }

  const promptsDir = join(import.meta.dir, "..", "prompts");

  // Config subcommand — non-interactive
  if (argv[0] === "config") {
    try {
      runConfigCommand(argv.slice(1));
    } catch (err) {
      process.stderr.write(`Error: ${(err as Error).message}\n`);
      process.exit(1);
    }
    process.exit(0);
  }

  // Inspect subcommand — non-interactive
  if (argv[0] === "inspect") {
    try {
      runInspectCommand(argv.slice(1), promptsDir);
    } catch (err) {
      process.stderr.write(`Error: ${(err as Error).message}\n`);
      process.exit(1);
    }
    process.exit(0);
  }

  // --- Main pipeline: parse → load → resolve → assemble → spawn ---
  let parsed;
  try {
    parsed = parseCliArgs(argv);
  } catch (err) {
    process.stderr.write(`Error: ${(err as Error).message}\n`);
    process.exit(1);
  }

  let loadedConfig;
  try {
    loadedConfig = loadConfig();
  } catch (err) {
    process.stderr.write(`Error: ${(err as Error).message}\n`);
    process.exit(1);
  }

  let config;
  try {
    config = resolveConfig(parsed, loadedConfig);
  } catch (err) {
    process.stderr.write(`Error: ${(err as Error).message}\n`);
    process.exit(1);
  }

  const env = detectEnv();
  const templateVars = buildTemplateVars(env);
  const prompt = assemblePrompt({ mode: config, templateVars, promptsDir });

  // --print: output the assembled prompt and exit
  if (parsed.modifiers.print) {
    process.stdout.write(prompt);
    process.exit(0);
  }

  // Write prompt to temp file
  const tempFile = writeTempPrompt(prompt);

  // Build claude args
  const claudeArgs: string[] = ["--system-prompt-file", tempFile];
  if (parsed.forwarded.appendSystemPrompt) {
    claudeArgs.push("--append-system-prompt", parsed.forwarded.appendSystemPrompt);
  }
  if (parsed.forwarded.appendSystemPromptFile) {
    claudeArgs.push("--append-system-prompt-file", parsed.forwarded.appendSystemPromptFile);
  }
  claudeArgs.push(...parsed.passthroughArgs);

  // Spawn claude with inherited stdio for full TUI handoff
  const proc = Bun.spawn(["claude", ...claudeArgs], {
    stdio: ["inherit", "inherit", "inherit"],
  });

  // Forward termination signals to child
  const forwardSignal = (sig: NodeJS.Signals) => proc.kill(sig);
  process.on("SIGINT", () => forwardSignal("SIGINT"));
  process.on("SIGTERM", () => forwardSignal("SIGTERM"));

  // Wait for claude to exit, propagate its exit code
  const exitCode = await proc.exited;

  // Clean up temp file (best-effort)
  try {
    const { rmSync } = await import("node:fs");
    const { dirname } = await import("node:path");
    rmSync(dirname(tempFile), { recursive: true, force: true });
  } catch { /* ignore */ }

  process.exit(exitCode ?? 1);
}

main();
```

**Implementation Notes**:
- `main()` is now `async` because `proc.exited` returns a Promise
- The key difference from `build-prompt.ts`: instead of printing a command string, this spawns `claude` directly with `Bun.spawn` and inherited stdio
- `build-prompt.ts` is preserved as-is — it's still useful for `--print` testing and for the bash wrapper during development. But `cli.ts` is the new canonical entry point.
- Temp file cleanup happens after claude exits. The bash wrapper's `trap` is replaced by cleanup code after `await proc.exited`.
- Signal forwarding ensures Ctrl+C reaches claude. `SIGINT` is the critical one — it's what Ctrl+C sends.
- `import.meta.dir` in compiled binary points to the binary's directory. Since prompts are embedded, this only matters for disk fallback.

**Acceptance Criteria**:
- [ ] `bun src/cli.ts create` launches Claude with the correct system prompt and full TUI
- [ ] Ctrl+C cleanly terminates both claude and the wrapper
- [ ] `/exit` in claude cleanly exits with code 0
- [ ] `--print` outputs the prompt to stdout and exits (no spawn)
- [ ] `--help` prints usage and exits
- [ ] `config` and `inspect` subcommands work as before
- [ ] Passthrough args (e.g. `-- --verbose`) reach claude
- [ ] Temp file is cleaned up after claude exits

### Unit 5: Version Bump Script

**File**: `scripts/bump-version.ts`

```typescript
#!/usr/bin/env bun
/**
 * Bumps version in package.json, commits, tags, and pushes.
 * The tag push triggers the release workflow.
 *
 * Usage: bun scripts/bump-version.ts <patch|minor|major|x.y.z>
 */
import { readFileSync, writeFileSync } from "node:fs";
import { execSync } from "node:child_process";
import { join } from "node:path";

function main(): void {
  const arg = process.argv[2];
  if (!arg) {
    process.stderr.write("Usage: bun scripts/bump-version.ts <patch|minor|major|x.y.z>\n");
    process.exit(1);
  }

  const pkgPath = join(import.meta.dir, "..", "package.json");
  const pkg = JSON.parse(readFileSync(pkgPath, "utf8"));
  const current: string = pkg.version;

  let next: string;
  if (/^\d+\.\d+\.\d+$/.test(arg)) {
    next = arg;
  } else {
    const [major, minor, patch] = current.split(".").map(Number);
    switch (arg) {
      case "patch": next = `${major}.${minor}.${patch + 1}`; break;
      case "minor": next = `${major}.${minor + 1}.0`; break;
      case "major": next = `${major + 1}.0.0`; break;
      default:
        process.stderr.write(`Invalid bump: "${arg}". Use patch, minor, major, or x.y.z\n`);
        process.exit(1);
    }
  }

  // Update package.json
  pkg.version = next;
  writeFileSync(pkgPath, JSON.stringify(pkg, null, 2) + "\n");

  // Regenerate embedded prompts (ensures they're fresh for the tagged commit)
  execSync("bun scripts/generate-prompts.ts", { stdio: "inherit" });

  // Stage, commit, tag, push
  execSync("git add package.json src/embedded-prompts.ts", { stdio: "inherit" });
  execSync(`git commit -m "Release v${next}"`, { stdio: "inherit" });
  execSync(`git tag v${next}`, { stdio: "inherit" });
  execSync(`git push && git push --tags`, { stdio: "inherit" });

  process.stdout.write(`Released v${next}\n`);
}

main();
```

**Implementation Notes**:
- Follows skilltap pattern: bump → commit → tag → push
- Tag push triggers the release workflow (Unit 7)
- Regenerates embedded prompts before committing so the tagged commit has fresh prompts
- No runtime deps — uses node:fs and node:child_process

**Acceptance Criteria**:
- [ ] `bun scripts/bump-version.ts patch` bumps 0.1.0 → 0.1.1
- [ ] `bun scripts/bump-version.ts minor` bumps 0.1.0 → 0.2.0
- [ ] `bun scripts/bump-version.ts major` bumps 0.1.0 → 1.0.0
- [ ] `bun scripts/bump-version.ts 2.0.0` sets exact version
- [ ] Creates git commit with message "Release v{version}"
- [ ] Creates git tag `v{version}`
- [ ] Pushes commit and tag
- [ ] Fails with clear error for invalid input

### Unit 8: curl-pipe-bash Install Script

**File**: `install.sh`

```bash
#!/bin/sh
set -e

# claude-mode installer
# Usage: curl -fsSL https://raw.githubusercontent.com/nklisch/claude-code-modes/main/install.sh | sh

REPO="nklisch/claude-code-modes"
INSTALL_DIR="${CLAUDE_MODE_INSTALL:-$HOME/.local/bin}"
BINARY_NAME="claude-mode"

main() {
  # Detect OS
  os="$(uname -s | tr '[:upper:]' '[:lower:]')"
  case "$os" in
    linux) os="linux" ;;
    darwin) os="darwin" ;;
    *) echo "Unsupported OS: $os" >&2; exit 1 ;;
  esac

  # Detect architecture
  arch="$(uname -m)"
  case "$arch" in
    x86_64|amd64) arch="x64" ;;
    aarch64|arm64) arch="arm64" ;;
    *) echo "Unsupported architecture: $arch" >&2; exit 1 ;;
  esac

  artifact="${BINARY_NAME}-${os}-${arch}"

  # Get latest release tag
  tag=$(curl -fsSL "https://api.github.com/repos/${REPO}/releases/latest" | grep '"tag_name"' | sed 's/.*"tag_name": *"//;s/".*//')
  if [ -z "$tag" ]; then
    echo "Error: could not determine latest release" >&2
    exit 1
  fi

  url="https://github.com/${REPO}/releases/download/${tag}/${artifact}"
  echo "Downloading ${artifact} ${tag}..."

  # Create install directory
  mkdir -p "$INSTALL_DIR"

  # Download binary
  curl -fsSL "$url" -o "${INSTALL_DIR}/${BINARY_NAME}"
  chmod +x "${INSTALL_DIR}/${BINARY_NAME}"

  # macOS: remove quarantine attribute
  if [ "$os" = "darwin" ]; then
    xattr -d com.apple.quarantine "${INSTALL_DIR}/${BINARY_NAME}" 2>/dev/null || true
  fi

  echo "Installed ${BINARY_NAME} to ${INSTALL_DIR}/${BINARY_NAME}"

  # Check if install dir is in PATH
  case ":$PATH:" in
    *":${INSTALL_DIR}:"*) ;;
    *)
      echo ""
      echo "Add ${INSTALL_DIR} to your PATH:"
      echo "  export PATH=\"${INSTALL_DIR}:\$PATH\""
      echo ""
      echo "Or add to your shell profile:"
      for profile in ~/.bashrc ~/.zshrc ~/.profile; do
        [ -f "$profile" ] && echo "  echo 'export PATH=\"${INSTALL_DIR}:\$PATH\"' >> $profile"
      done
      ;;
  esac
}

main
```

**Implementation Notes**:
- POSIX sh for maximum portability (no bash-isms)
- Installs to `~/.local/bin` by default, overridable via `$CLAUDE_MODE_INSTALL`
- Does NOT auto-modify shell profiles — prints instructions instead. This is less aggressive than skilltap's approach but safer.
- Uses GitHub API to find latest release tag, then downloads the correct binary
- macOS quarantine removal prevents "unidentified developer" warnings

**Acceptance Criteria**:
- [ ] Detects linux/darwin and x64/arm64 correctly
- [ ] Downloads the correct binary for the platform
- [ ] Binary is executable after install
- [ ] Prints PATH instructions if install dir not in PATH
- [ ] Fails with clear error on unsupported OS/arch
- [ ] `CLAUDE_MODE_INSTALL` overrides install directory

## Implementation Order

1. **Unit 1**: Generate Embedded Prompts Map — foundation for everything else
2. **Unit 2**: Refactor `assemble.ts` — makes prompt assembly work with embedded map
3. **Unit 3**: TypeScript CLI entry point (`cli.ts`) — replaces bash wrapper
4. **Unit 4**: Update `package.json` — wires up bin, files, scripts
5. **Unit 9**: .gitignore housekeeping — before first build
6. **Unit 5**: Version bump script — needed before first release
7. **Unit 7**: CI workflow — should be in place before release workflow
8. **Unit 6**: Release workflow — ties it all together
9. **Unit 8**: Install script — can be written and tested independently once release workflow is running

Units 1-4 are tightly coupled and should be implemented as a sequence. Units 5-9 are independent and can be done in any order after 1-4.
